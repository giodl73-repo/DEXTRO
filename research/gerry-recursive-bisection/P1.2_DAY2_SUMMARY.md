## Root Cause Analysis

### Hypothesis 1: Target Weights Not Being Used
**Issue**: METIS may not be respecting the 2D target_weights for multi-constraint partitioning

**Evidence**:
- Target weights specify 55% minority concentration in 2 districts
- Actual results show even distribution (~35-43% across all districts)
- Looks like METIS is ignoring the second constraint

**Investigation Needed**:
- Check if pymetis/gpmetis.exe supports multi-constraint with target weights
- Verify METIS graph file format is correct for multi-constraint
- Test with simpler 2-district case to isolate issue

### Hypothesis 2: METIS Balancing Constraints Equally
**Issue**: METIS might be treating both constraints with equal priority, leading to over-balancing of minority VAP

**Evidence**:
- Multi-constraint typically tries to balance ALL constraints
- We want: Concentrate minority in 2 districts, distribute in others
- METIS may be: Balance minority evenly across all districts

**Solution**:
- May need to use ubvec (per-constraint imbalance tolerance)
- Allow higher imbalance on minority constraint
- Or use different approach: post-processing optimization

### Hypothesis 3: Geometric Constraints Dominate
**Issue**: Alabama's geography may prevent 55%+ districts even with VRA awareness

**Evidence**:
- Black population concentrated in "Black Belt" but not enough for 2 districts
- Compactness + contiguity may limit minority concentration
- Baseline already struggles at 46%

**Test**: Try on state with clearer minority geography (e.g., Georgia, Maryland)

## Next Steps

### Option A: Fix Multi-Constraint METIS (2-3 days)
**Approach**: Debug and tune METIS multi-constraint parameters
- Verify pymetis multi-constraint API usage
- Test ubvec (per-constraint imbalance factors)
- Try k-way partitioning with constraints
- Test on multiple states to validate

**Pros**: Proper algorithmic solution, reusable
**Cons**: May hit METIS limitations, time-consuming

### Option B: Post-Processing Optimization (1-2 days)
**Approach**: Run baseline algorithm, then adjust boundaries to create MM districts
- Start with baseline compact districts
- Identify districts closest to 50% threshold
- Swap boundary tracts to push above 50%
- Minimize compactness loss

**Pros**: Faster, more control, guaranteed to work
**Cons**: Two-step process, not pure algorithmic

### Option C: Hybrid: Seeded Initial Partition (2-3 days)
**Approach**: Pre-identify MM district cores, then partition around them
- Cluster tracts with high minority % into MM district seeds
- Run standard METIS on remaining tracts
- Ensures MM districts while maintaining compactness elsewhere

**Pros**: Balanced approach, cleaner than post-processing
**Cons**: Requires seed selection algorithm
