# Review 5 — Christina Liang
**Paper**: F.2: NestSection at Scale — Spine-Compatible Bicameral Redistricting for All 50 States
**Round**: R1
**Score**: 3/4

## Summary

F.2 applies NestSection to 42 compatible bicameral states. Reproducibility questions focus on (1) whether the gcd values in Table 1 are independently verifiable, (2) whether the Kuriwaki precinct data used for partisan analysis is available and properly attributed, and (3) whether the 42-state results can be reproduced from the described infrastructure.

## Strengths

The gcd values in Table 1 are, in principle, derivable from public records: each state's chamber sizes are set by state constitution or statute and are publicly available from the NCSL (National Conference of State Legislatures). Spot-checking: WA (98, 49, gcd=49), MN (134, 67, gcd=67), OH (99, 33, gcd=33), FL (120, 40, gcd=40), CA (80, 40, gcd=40), NH (400, 24, gcd=8) — all correct by independent calculation. The table is reproducible from public data.

The Kuriwaki precinct data citation is appropriate: the 2023 paper provides precinct-level election returns interpolated to census blocks, which can be reaggregated to block groups for the partisan analysis. The methodology for the partisan comparison (comparing independent vs. NestSection seat counts using the same precinct data) is described clearly enough to be reproduced.

## Concerns

**C1 — Block-group prerequisite for NestSection.** F.2 states that WA uses block-group resolution (H=98, S=49, g=49, resolution block_group). This means that each of the 49 spine super-regions contains approximately 4,874/49 ≈ 99 block groups and must be redistricted into 2 house and 1 senate district. This is a block-group-level spine redistricting problem. The paper does not specify whether the block-group adjacency data required for the spine redistricting is the same file generated by F.1's preprocessing, or whether a separate step is needed to generate spine-constrained adjacency graphs.

**C2 — 12 states without precinct data not identified.** The partisan analysis covers 30 of 42 compatible states. The 12 excluded states are not named. A reproducibility reviewer cannot verify which states are in the 30-state sample without this information.

**C3 — Senate balance verification.** Table 2 reports that NestSection achieves ≤0.5% maximum deviation in both chambers for all 42 compatible states. This is the key feasibility claim of the paper. However, the paper does not provide a full 42-state verification table — it shows only 7 states in Table 2. The claim "all 42 compatible states succeed" requires either a complete appendix table or a pointer to a repository where the full results are available.

**C4 — NestSection flag not in confirmed production state.** The CLI usage in Section 2.3 specifies --chamber nest as the flag for NestSection mode. The CLAUDE.md for the project does not list --chamber nest as an existing production flag — the documented flags are --chamber house and --chamber senate. If --chamber nest has not yet been implemented in the Rust binary, then the F.2 results were generated by a different mechanism (possibly Python) and the CLI examples are forward-looking rather than current. This should be clarified.

## Recommendation

Accept with revisions. The gcd table is reproducible and the methodology is sound. C4 (NestSection CLI flag production status) must be clarified, as it affects whether the described computational approach is actually what produced the results.
