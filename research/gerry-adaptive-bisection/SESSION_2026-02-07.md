## Key Concepts Introduced

### Adaptive Bisection Algorithm

**Core Principle**: At each split creating k_L and k_R districts, evaluate both orderings (k_L, k_R) and (k_R, k_L), choose the one producing higher minority concentration.

**Evaluation Function**:
```
m_max(G_i) = min(Σ m_v · w_pop(v) / target_district_pop, 1.0)
```

**Scoring Function**:
```
score(G_L, G_R) = 1[m_max(G_L) ≥ τ] + 1[m_max(G_R) ≥ τ] + 0.01·(m_max(G_L) + m_max(G_R))
```

Prioritizes splits enabling MM districts (discrete) with continuous tie-breaker.

### Theoretical Framework

**Why Adaptive Helps**:
- Eliminates arbitrary predetermined structure
- Approximates optimal tree T* through greedy local optimization
- Achieves ≥ expected performance of random tree selection

**Why Adaptive Insufficient**:
- Constrained by binary tree structure (once split, vertices never interact across boundary)
- Local optimization vs n-way's global optimization
- Explores O(k²) partitions vs n-way's O(k^|V|) space

**Gap Magnitude**: 1.2 points (adaptive 46.1% vs n-way 47.3%)
- Often substantively irrelevant (both achieve VRA targets)
- Critical in borderline states (45-49% concentration)

### Method Selection Framework

| Method | Transparency | VRA Performance | Computational Cost | When to Use |
|--------|-------------|-----------------|-------------------|-------------|
| Predetermined | High | Low | 1× | Never (inferior) |
| Adaptive | High | High | 3-5× | Transparency critical |
| N-way | Low | Highest | 2-3× | Default choice |
